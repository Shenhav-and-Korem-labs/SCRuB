SCRuB: Source-tracking for Contamination Removal
     in microBiomes

We are currently in beta testing! Please report any issues you faced while using SCRuB in our Issues Page, or email gia2105@columbia.edu .

Welcome to SCRuB! This is a software package designed to provide researchers with a state of the art tool to remove contamination from microbial datasets.

In addition to the statistical framework of SCRuB, we provide preSCRuB, a personalized interface to offer user recommendations to optimize negative control strategies during study design processes. Effective use of preSCRuB will allow for even stronger decontamination using SCRuB in future analyses.

Installation

SCRuB is currently available through our github repository, which can be installed using devtools:

devtools::install_github("korem-lab/SCRuB")
install_torch()

Additionally, we provide SCRuB as a QIIME2 plugin, which can be installed from a Qiime2 environment using pip:

pip install git+https://github.com/korem-lab/q2-SCRuB.git

FAQs

Why SCRuB?

SCRuB offers the advantage of combing source-tracking approaches with contamination source modelling, while tracking for possible sources of well-to-well leakage. Thus, it provides an effective and low-risk approach to microbial decontamination, which has been shown to improve phenotype prediction in downtream applications.
